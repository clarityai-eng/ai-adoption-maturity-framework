# **AI Coding Maturity – Scoring Rubric (Team Level)**

**Scale (used everywhere)**  
1 = Not present  
2 = Ad hoc / individual  
3 = Emerging / inconsistent  
4 = Established / consistent  
5 = Advanced / optimized


## **1\. Adoption & Frequency**

**How widely and consistently AI coding tools are used**

| Score | Description |
| ----- | ----- |
| 1 | No usage or explicitly avoided |
| 2 | One or two individuals experiment occasionally |
| 3 | Several members use AI weekly for specific tasks |
| 4 | Majority use AI daily or near-daily |
| 5 | AI assistance is a default part of development |

## **2\. Use Case Breadth**

**Range of coding activities supported by AI**

| Score | Description |
| ----- | ----- |
| 1 | No meaningful use cases |
| 2 | Narrow (e.g. snippets, syntax help) |
| 3 | Multiple use cases (coding, debugging, tests) |
| 4 | End-to-end coverage (design → delivery) |
| 5 | Creative and strategic use (architecture exploration, trade-offs) |

## **3\. Tooling & Integration**

**Quality of tools and how well they fit workflows**

| Score | Description |
| ----- | ----- |
| 1 | No tools or blocked by policy |
| 2 | Informal / shadow usage |
| 3 | Approved tools, basic IDE integration |
| 4 | Well-integrated, consistent tooling |
| 5 | Optimized setup, custom workflows, shared configs |

## **4\. Skill & Prompting Maturity**

**Ability to use AI effectively, not just frequently**

| Score | Description |
| ----- | ----- |
| 1 | Don’t know how to use tools |
| 2 | Trial-and-error prompting |
| 3 | Reasonable prompts, iterative refinement |
| 4 | Advanced prompting, context management |
| 5 | Systematic approaches, reusable patterns |

## **5\. Quality Control & Safety**

**How AI-generated code is validated and governed**

| Score | Description |
| ----- | ----- |
| 1 | Blind trust or no usage |
| 2 | Manual checking, inconsistent |
| 3 | Explicit review expectations |
| 4 | Clear rules, security and quality awareness |
| 5 | Strong safeguards, AI-aware review standards |

## **6\. Team Alignment & Culture**

**Shared norms vs. individual behavior**

| Score | Description |
| ----- | ----- |
| 1 | Fragmented, polarized views |
| 2 | Silent divergence (some use, some resist) |
| 3 | Open discussion, no clear norms |
| 4 | Shared expectations and encouragement |
| 5 | Strong learning culture, peer enablement |

## **7\. Champions & Knowledge Sharing**

**Presence and effectiveness of pioneers**

| Score | Description |
| ----- | ----- |
| 1 | No champions |
| 2 | Isolated power users |
| 3 | Informal sharing |
| 4 | Active champions influencing squad |
| 5 | Champions scale practices beyond squad |

## **8\. Impact & Outcomes**

**Observable value from AI coding**

| Score | Description |
| ----- | ----- |
| 1 | No impact or negative impact |
| 2 | Anecdotal, unclear benefits |
| 3 | Clear productivity gains |
| 4 | Measurable improvements (speed, quality) |
| 5 | Structural impact on delivery model |

# **Overall Maturity Levels**

**Calculate:**  
Average of all dimensions (or weight Adoption, Quality, and Impact higher if desired).

| Level | Range | Interpretation |
| ----- | ----- | ----- |
| Level 1 | 1.0 – 1.9 | Unaware / Resistant |
| Level 2 | 2.0 – 2.4 | Experimental |
| Level 3 | 2.5 – 3.4 | Emerging |
| Level 4 | 3.5 – 4.4 | Established |
| Level 5 | 4.5 – 5.0 | Leading / Transformational |

## **Red Flags to Note (Qualitative)**

* High adoption \+ low quality control  
* Strong pioneers \+ strong detractors  
* Heavy usage outside approved tooling  
* Over-reliance for complex logic

## **Recommended Outputs per Squad**

* Maturity level  
* Top 2 strengths  
* Top 2 gaps  
* Named champions  
* 1–2 concrete next actions
